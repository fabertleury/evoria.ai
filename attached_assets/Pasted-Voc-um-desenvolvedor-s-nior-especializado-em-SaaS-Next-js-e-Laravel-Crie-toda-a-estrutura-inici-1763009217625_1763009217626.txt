Você é um desenvolvedor sênior especializado em SaaS, Next.js e Laravel. Crie toda a estrutura inicial de um sistema chamado "Momentia", um micro SaaS que permite ao anfitrião de um evento gerar um QR Code para que os convidados enviem vídeos curtos (stories) de um evento.

O sistema deve seguir a arquitetura abaixo:

Frontend: React com Next.js (configurado como PWA)
Backend: Laravel (PHP 10+) com APIs REST e WebSocket
Banco de dados: PostgreSQL
Storage: AWS S3 ou Supabase Storage (compatível com S3)
Realtime: Pusher / Supabase Realtime / Socket.IO
Workers: BullMQ ou RabbitMQ (para processamento assíncrono)
Pagamentos: Stripe + Pix (via API brasileira). Implementar endpoints de webhook para reconciliar compras.
Infraestrutura: Docker Compose com containers separados (frontend, backend, banco e fila)
Autenticação: Sanctum no Laravel e JWT no frontend

Funcionalidades que devem ser consideradas:

1. Cadastro e login do anfitrião.
2. Criação de eventos com título, data e capa.
3. Geração automática de QR Code e link exclusivo para o evento.
4. Página pública para convidados enviarem vídeos curtos (máx. 30s), digitando apenas nome e enviando uma selfie.
5. Armazenamento seguro dos vídeos no S3/Supabase.
6. Sistema de moderação:
   - IA (simulada) detecta nudez ou conteúdo indevido e bloqueia o vídeo automaticamente.
   - O anfitrião tem um link privado para moderar vídeos manualmente (aprovar, excluir, pular).
7. Módulo “Telão Ao Vivo”:
   - Exibe os vídeos em tempo real em formato de stories.
   - Permite que convidados usem créditos para aparecer no telão.
   - Convidados podem comprar créditos (R$5 = 5 créditos).
   - “Skip” para furar fila (R$2) e entrar no telão imediatamente.
   - Controle de tempo no telão (30s por vídeo).
   - Backend gerencia transações e saldo de créditos.
8. Painel do anfitrião:
   - Estatísticas em tempo real (número de vídeos, visualizações, créditos vendidos, etc.)
   - Botão para ativar/desativar o “Telão Ao Vivo”.
9. Pagamentos:
   - Stripe (cartão e Apple Pay / Google Pay).
   - Pix integrado (checkout simples).
   - Webhook para atualizar status das compras.
10. Segurança:
   - Upload validado por tipo e tamanho.
   - Moderação de conteúdo antes da publicação.
   - Logs de evento (envios, exclusões, bloqueios).

Entrega esperada:
- Estrutura completa do projeto em monorepo (frontend + backend).
- Configuração do Docker Compose (containers: web, api, db, redis/fila).
- Scripts de setup inicial e seed de dados (exemplo de evento e vídeos de teste).
- Boas práticas de arquitetura (services, controllers, events, queues, jobs).
- Documentação básica em README.md com instruções de instalação e rodar o ambiente.

Atenção:
- Não gerar lógica de IA (somente simular moderação).
- Foco em escalabilidade e boa organização de pastas.
- Utilizar TypeScript no frontend e boas práticas no backend (Repository Pattern e Events).
